# atlas-fastapi E2B template

Python 3.12 + FastAPI 0.115+ + Pydantic 2 + uvicorn + sqlalchemy + alembic + pytest + ruff + uv.

Used by Atlas's developer role when the architect's `canvasManifest.artifactKind === "backend-rest-api"` AND `ATLAS_FF_MULTI_STACK=true`.

## Pre-installed runtime deps

- **fastapi** 0.115+ — async web framework
- **pydantic** 2.x + **pydantic-settings** — typed schemas + env config
- **uvicorn[standard]** — ASGI dev server
- **httpx** — async HTTP client
- **sqlalchemy** 2.x + **alembic** + **psycopg2-binary** — ORM, migrations, Postgres driver
- **pytest** + **pytest-asyncio** — test framework
- **ruff** — lint + format
- **uv** — package manager + venv

## Out-of-the-box endpoints

- `GET /` — app metadata (name, version)
- `GET /health` — `{"status": "ok", "stack": "fastapi", "atlas": "sandbox-ready"}`
- `GET /docs` — Swagger UI (auto-generated by FastAPI)
- `GET /openapi.json` — OpenAPI spec

## Local smoke test (no E2B credit)

```bash
cd packages/sandbox-e2b/templates/atlas-fastapi
./scripts/smoke-test-local.sh
```

## Build + push to E2B

```bash
cd packages/sandbox-e2b/templates/atlas-fastapi
export E2B_API_KEY=e2b_...
./scripts/build-template.sh
# Capture the printed template ID; add it to e2b.toml's template_id; commit.
```

## Wire into atlas-web

When `ATLAS_FF_MULTI_STACK=true` AND architect classifies the project as `backend-rest-api`, the sandbox factory routes provisioning to this template automatically. Per-project override via `ATLAS_DEFAULT_SANDBOX_TEMPLATE=atlas-fastapi`.
